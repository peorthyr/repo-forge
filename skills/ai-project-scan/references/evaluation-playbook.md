# Scan Evaluation Playbook

## 1. Define scenarios
Use at least 3 scenarios:
- partially structured project
- legacy noisy docs
- already clean architecture package

## 2. Run with baseline
Compare with-skill and without-skill reports.

## 3. Objective checks
Validate that reports include:
- inventory of architecture files
- compatibility status
- recommendation (keeper/seed/no-op)
- explicit rationale

## 4. Human review
Check if recommendations are practical and conservative.

## 5. Iterate
Refine scan heuristics and rerun scenarios.
