# Routing Matrix

## do-now
Use when request is small, local, clear, and reversible.

## plan
Use when request is multi-step, risky, cross-surface, or needs verification.
Destination: `plans/`

## backlog
Use when request is valid but not immediate.
Destination: `BACKLOG.md`

## idea
Use when the request is promising but immature.
Destination: idea section or equivalent.

## brainstorm
Use when options are open and exploration is needed before committing.
Destination: `docs/BRAINSTORM.md` or equivalent.

## adr-candidate
Use when the request changes architecture, conventions, policy, or stable direction.
Destination: ADR candidate or `adr/`

## analysis-first
Use when impact, compatibility, or current state is unclear.
Destination: analysis note or `ai-project-scan`

## defer
Use when request is valid but lower priority.
Destination: backlog or idea parking.

## reject
Use when request is incoherent, harmful, out of scope, or too costly for current priorities.
